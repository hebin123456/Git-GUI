# GitForkLike 用户手册

面向在本机使用 Electron 版 GitForkLike 的简要说明。界面语言可在「设置 → 外观」中切换简体中文 / English。

## 1. 基本操作

- **打开仓库**：菜单「文件 → 打开仓库…」，或启动后按提示选择目录。
- **克隆**：「文件 → 克隆仓库…」，填写远程 URL 与本地父目录。
- **多标签**：可同时打开多个仓库，标签栏切换；工作区状态（打开的标签等）保存在本机，与 Git 仓库文件无关。
- **刷新**：「文件 → 刷新状态」或工具栏相应按钮，重新读取 `git status`、分支与历史等。

## 2. 变更视图（Changes）

- **未暂存 / 已暂存**：左侧文件树区分工作区与索引；可单文件或批量暂存、取消暂存。
- **差异预览**：选中文件后右侧显示 diff；可在「差异选项」中切换左右分栏 / 单列、上下文行数、忽略空白等（默认值在「设置 → 差异」中配置）。
- **二进制文件**：检测到二进制内容时，不再把原始 diff 当作文本展示（避免乱码），改为占位提示；仍可在列表中看到该文件及状态。
- **合并冲突**：冲突文件在列表中有醒目标记；可采用「我方 / 对方」写入工作区，或点击 **Mergetool** 调用外部合并工具（见下文「合并工具」）。
- **提交**：填写主题与可选说明，勾选「修订上次提交」可进行 amend（需注意与远程历史是否已推送）。

## 3. 历史与提交详情

- **所有提交**：时间线或列表浏览提交；可搜索、加载更多（单次加载条数上限在「设置 → 性能」中调整）。
- **选中提交**：可查看元信息、与父提交的差异等。
- **提交详情的文件树**：列出**该次提交相对其父提交（或根提交的 `--root`）的全部变更文件**，目录结构便于浏览；合并提交等场景下与 `git diff-tree` 一致，避免遗漏文件。

## 4. 打开资源管理器与 Git 命令行

- **资源管理器**：顶栏「打开 → 用资源管理器打开」在仓库根目录打开系统文件管理器。
- **Git 命令行**：「打开 → 在 Git 命令行中打开」在仓库根目录启动终端；若未在设置中指定程序，Windows 下会按内置顺序尝试（如 Git Bash、Windows Terminal、cmd）。
- **子模块**：在侧栏子模块条目上可选择在资源管理器或**终端**中打开；终端行为同样遵循「设置 → 工具」中的 **Git 命令行终端**路径。

### 4.1 第三方终端（设置 → 工具）

在 **Git 命令行终端**中填写终端可执行文件的**完整路径**（例如 `wt.exe`、PowerShell、`cmd.exe` 等）。保存设置后，上述「在 Git 命令行中打开」及子模块「在终端中打开」均会使用该程序；留空则恢复内置检测顺序。

部分功能（如「部分暂存」「分段贮藏」「交互式变基起点」）会在仓库根目录打开该终端并预填一条 `git` 命令，同样使用此处配置的终端。

## 5. 合并工具（Beyond Compare / WinMerge）

在 **设置 → 工具** 中：

1. **合并冲突工具**：选择「跟随 Git 配置」或预设 **Beyond Compare 4 / 3**、**WinMerge**。
2. **合并工具可执行文件**：若对应程序已加入系统 `PATH`，可留空；否则填写例如 `BComp.exe`、`WinMergeU.exe` 的完整路径。

在「变更」视图中对冲突文件点击 **Mergetool** 时，应用会调用 `git mergetool`（非 default 预设时会通过 `-c mergetool.<tool>.path=` 传入路径）。完成后请按工具提示保存，并在应用中刷新状态。

若仍无法启动，可在本机终端执行 `git config merge.tool` 检查全局配置，或与上述设置中的预设、路径对照排查。

## 6. 设置项摘要

| 区域 | 说明 |
|------|------|
| 外观 | 主题、界面语言 |
| 差异 | 默认分栏方式、上下文行数、忽略空白等 |
| 工具 | 自定义 Git 终端路径、合并工具预设与可执行路径 |
| 性能 | 「所有提交」单次加载的提交数量上限 |
| 数据 | 清除工作区记录（不影响磁盘上的仓库与已保存的「设置」） |

修改后请点击「保存」写入本机（与主题等一同保存在浏览器 localStorage 中）。

## 7. 常见问题

- **终端一闪而过或报错**：检查「工具」中的路径是否正确、该终端是否支持以工作目录启动；Windows Terminal 通常使用 `wt.exe`。
- **Mergetool 无窗口**：确认已安装对应软件、路径或 PATH 正确；可先在同一仓库下于命令行执行 `git mergetool` 做对比。
- **大仓库卡顿**：适当降低「性能」中的历史条数，在历史视图中需要时再点「加载更多」。
